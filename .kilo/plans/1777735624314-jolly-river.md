# ProjectStoryView Implementation Plan

## Overview
Implement ProjectStoryView - a narrative, read-only timeline view that presents a project's journey as a story, organized by phases (chapters) with visual density representation of sessions, milestones, and inferred drift/flow patterns.

## Key Requirements from Brief

### What It Is
- Narrative representation of a single project
- Chapter-based view organized by Phases
- Timeline from first recorded session to most recent
- Read-only reflective surface ("closer to opening a book than a dashboard")
- Replaces main content area when project selected

### What It Is Not
- NOT a productivity report or metrics dashboard
- NOT a configuration/editing screen
- NOT a real-time coaching interface
- NOT for micro-optimization

### Core Philosophy
- "Nothing appears here unless it helps future-you remember or reckon"
- Shows patterns, momentum, drift, turning points
- Infers drift/flow visually (gaps, density changes, phase shifts)
- No explicit labels for drift/flow - inferred through visual design

### Emotional Goals
- Feel weight of time
- Notice forgotten patterns
- Mild discomfort at drift
- Pride at momentum/completion
- Sense of clear chapters

## Architecture

### New Components to Create

1. **ProjectStoryViewModel.swift** (ViewModel)
   - Loads all sessions for a project
   - Groups sessions by phaseID (chronologically)
   - Derives:
     - Phase spans (first/last session date per phase)
     - Session density (count and temporal distribution)
     - Temporal gaps (time between sessions)
     - Milestone positions
     - Drift/flow indicators
   - No mutations, no writes - pure read-layer
   - @MainActor ObservableObject

2. **ProjectStoryView.swift** (View)
   - Main view file
   - Timeline-based vertical scroll view
   - Project header (name, emoji, color, start date)
   - Phase sections (chapters) with date ranges
   - Visual session density within each phase
   - Milestone markers (star icon from isMilestone sessions)
   - Visual gap representation for drift
   - Minimal controls (back/close button)

3. **ProjectStoryRow.swift** (optional View component)
   - Visual representation of session density within a phase
   - Could be bar/symbol-based

4. **MilestoneMarker.swift** (optional View component)
   - Star/icon with action text
   - Canonised, distinctive appearance

### Integration Points

**Sidebar Navigation:**
- `SidebarStateManager` needs new case: `.projectStory(Project)`
- Accessed via button on project row in Projects tab
- Replaces main content area (like SessionsView does currently)
- Sidebar remains for navigation

**Data Loading:**
- Uses `SessionManager.shared.allSessions` (already loaded)
- Uses `ProjectManager.shared.loadProjects()` for project metadata
- Listens to `.projectsDidChange`, `.sessionDidEnd`, `.sessionsDidLoad` notifications

**File Dependencies:**
- No new file writes
- Read-only from:
  - `YYYY-data.csv` files (via SessionManager)
  - `projects.json` (via ProjectManager)

### ViewModel Responsibilities

```
ProjectStoryViewModel:
- project: Project
- phasesWithSessions: [(phase: Phase, sessions: [SessionRecord])]
- allProjectSessions: [SessionRecord] (sorted chronologically)
- milestones: [SessionRecord] (isMilestone == true)
- 
- phaseRanges: [Phase: (start: Date, end: Date)]
- sessionDensity: [Phase: density metric]
- temporalGaps: [(after: Date, duration: TimeInterval)]
- 
- loadProjectData(projectID: String) async
- computePhaseSpans()
- computeDensity()
- groupSessionsByPhase()
```

### UI/UX Design Details

**Overall Layout:**
- Full-width vertical scroll view
- Top: Project header (anchors the story)
- Middle: Phase sections (chapters) in chronological order
- Each phase section:
  - Phase name/identifier
  - Date range (e.g., "Jan 15 - Mar 22")
  - Visual session density indicator
  - Gap indicators between phases
  - Milestone markers positioned chronologically

**Visual Design:**
- Typography: Serious, book-like (not dashboard-like)
- Color: Project color used sparingly (accents, not overwhelming)
- Density: Could use bar height, dot density, or subtle background gradient
- Milestones: Star/bookmark icon with action text, canonised appearance
- Gaps: Lighter visual weight, more space, subtle separator
- Phase transitions: Clear chapter breaks

**Interaction:**
- Read-only (no buttons except close/back)
- Scroll to navigate timeline
- Minimal controls: close button to return to projects list
- No inline editing
- No sorting/filtering/reordering

### Data Flow

```
User selects project in ProjectsView
        ↓
SidebarStateManager.show(.projectStory(project))
        ↓
ContentView switches from SessionsView to ProjectStoryView
        ↓
ProjectStoryView created with view model
        ↓
ProjectStoryViewModel.loadProjectData(project.id)
        ↓
  - Fetch project from ProjectManager
  - Filter allSessions by projectID
  - Sort by startDate
  - Group by phaseID
  - Compute derived metrics
        ↓
View renders timeline
        ↓
Notifications (.projectsDidChange, .sessionDidEnd, .sessionsDidLoad) trigger refresh
```

### Edge Cases & Validation

1. **Project with no sessions:**
   - Show "No sessions yet" message
   - Still show project header

2. **Project with no phases defined:**
   - Group all sessions under "Unknown" or "General"
   - Still show timeline

3. **Archived project:**
   - Show normally (read-only view should show archived projects)
   - Maybe subtle "Archived" indicator

4. **Deleted project:**
   - SidebarState should handle gracefully
   - Return to ProjectsView

5. **Concurrent updates:**
   - ViewModel observes notifications
   - Reloads data on changes

6. **Large session count:**
   - Virtualized list (LazyVStack/LazyVGrid)
   - Density aggregation reduces rendering load

### Related Code References

**Pattern to follow:**
- `ProjectsViewModel.swift` - ObservableObject pattern, async loading
- `SessionsView.swift` - Grouped list with sections
- `SessionManager.swift` - Accessing session data
- `ProjectManager.swift` - Accessing project data

- Single project only (not all sessions)
- Phase-based grouping (not date-based)
- Density visualization (not individual rows)
- No editing/actions
- More narrative, less utilitarian
